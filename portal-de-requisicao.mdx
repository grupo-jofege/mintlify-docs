## Resumo geral

O `Requisições RH` é o projeto que vai centralizar a abertura, acompanhamento e aprovação de solicitações de RH em uma experiência própria, integrada ao ecossistema TOTVS.

O foco inicial do MVP é o fluxo de **Aumento de Quadro**, permitindo que a jornada aconteça no portal com rastreabilidade, integração com ACL/RM e base para evolução futura de novos processos.

## Objetivo do projeto

O projeto `Requisições RH` tem como objetivo criar um portal próprio para abertura, acompanhamento e aprovação de solicitações de RH, com foco inicial no fluxo de **Aumento de Quadro**.

O portal deve permitir que gestores e cenários delegados executem a jornada fora da experiência nativa do RM, mantendo integração com o ecossistema TOTVS e com o fluxo de aprovações já utilizado pela empresa.

## Escopo funcional atual

O foco atual do MVP é:

- autenticação do usuário
- abertura de solicitação de Aumento de Quadro
- persistência da solicitação no portal
- integração com ACL e RM
- início e acompanhamento do fluxo de aprovação
- listagem e detalhe da solicitação
- administração de RH Central e permissões

## Componentes principais da solução

### Portal RH

Responsável pela interface do usuário, fluxo das telas, formulário, listagem, detalhe e experiência de aprovação.

### API Portal RH

Responsável por regras próprias do portal, persistência, rastreabilidade, vínculo entre processos e orquestração das integrações.

### ACL

Responsável pela integração com o RM, incluindo aprovações, tickets e fluxos necessários.

### API de Login

Será utilizada uma nova rota na API `JofegeLogin` para atender o fluxo de autenticação necessário ao projeto.

## Pontos técnicos relevantes já definidos

- a autenticação será tratada por uma nova rota na API de login
- a ACL será usada como base para iniciar a aprovação
- ao iniciar a aprovação, haverá também uma etapa ligada ao processo de **Aumento de Quadro** no RM
- será necessário manter o vínculo entre aprovação e Aumento de Quadro
- haverá um renderizador de Markdown vinculado ao processo para montar conteúdo dinâmico na tela de detalhe da aprovação

## Recomendação de vínculo entre aprovação e Aumento de Quadro

A recomendação mais segura é manter esse vínculo no backend do Portal RH com persistência própria.

Exemplo de dados a manter:

- `idPortal`
- `idTicketRm`
- `idAumentoQuadroRm`
- `tipoSolicitacao`
- `statusPortal`
- `statusRm`
- `createdAt`
- `updatedAt`
