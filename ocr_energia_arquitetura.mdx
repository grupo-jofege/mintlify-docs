Esta página descreve como a solução foi organizada no `n8n`, quais são os componentes principais e como o fluxo atravessa captura, validação de vínculo, OCR, persistência e notificação.

## Visão geral

A solução substitui o processo Python legado por uma arquitetura em `n8n`, preservando a regra de negócio principal:

1. buscar documentos de energia no `V360`
2. registrar e controlar o documento em `CONTA_ENERGIA_DOCUMENTO`
3. validar previamente se o `ID_V360_DOCUMENTO` possui vínculo em `TMOVCOMPL.IDV360`
4. obter a URL válida do PDF
5. baixar o arquivo
6. enviar o PDF ao `Recognition`
7. fazer polling até conclusão
8. persistir o resultado tratado em `CONTA_ENERGIA_CONSUMO`
9. enviar o resumo da execução por e-mail

## Componentes

| Componente | Papel |
| --- | --- |
| \`OCREnergia | Orquestrador\` | coordena a execução, busca documentos, controla a fila, valida `IDMOV` e chama os subworkflows |
| \`OCREnergia | Download Arquivo\` | renova a URL assinada e realiza o download do PDF |
| \`OCREnergia | Recognition\` | envia o arquivo para OCR, faz polling, transforma o retorno e persiste os consumos |
| \`OCREnergia | Envio Email\` | consolida o resumo operacional e dispara a notificação final |

## Fluxo ponta a ponta

```text
Schedule/Manual
  -> Orquestrador
    -> Buscar novos documentos no V360
    -> Buscar documentos pendentes no banco
    -> Unificar fila
    -> Para cada documento
         -> Upsert em CONTA_ENERGIA_DOCUMENTO
         -> Validar IDMOV / CODCOLIGADA
         -> Se vínculo existir
              -> Download Arquivo
              -> Recognition
         -> Se vínculo não existir
              -> Atualizar erro operacional e seguir para o próximo item
    -> Envio Email
```

## Estratégia de estado

Etapas operacionais acompanhadas em `CONTA_ENERGIA_DOCUMENTO.ETAPA`:

- `ENCONTRADO`
- `IDMOV_ENCONTRADO`
- `BAIXANDO_PDF`
- `ARQUIVO_BAIXADO`
- `ENVIADO_RECOGNITION`
- `PROCESSANDO`
- `SUCESSO`

Campos de erro operacionais:

- `ETAPA_ERRO`
- `MENSAGEM_ERRO`
- `MOTIVO_ERRO`

## Decisões importantes

- URL do PDF não é persistida no banco
- polling do `Recognition` segue a lógica do legado: `5s`, `120` tentativas, timeout de `10 minutos`
- a vinculação operacional usa `GFILIAL` (somente `ATIVO = 1`) e `TMOVCOMPL`
