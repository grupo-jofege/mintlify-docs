## Visão geral

| Tabela | Finalidade |
| --- | --- |
| `CONTA_ENERGIA_DOCUMENTO` | controla o ciclo de vida do documento, estados, auditoria, OCR e falhas |
| `CONTA_ENERGIA_CONSUMO` | armazena os itens tratados do OCR prontos para análise, integração e BI |

## Regras de vinculação

O vínculo com `GFILIAL` (somente `ATIVO = 1`) e `TMOVCOMPL` usa o CNPJ do cliente e o `ID_V360_DOCUMENTO` para tentar localizar:

- `CODCOLIGADA`
- `IDMOV`

Exemplo conceitual:

```sql
SELECT
  TMOVCOMPL.IDMOV,
  FILIAL.CODCOLIGADA
FROM CONTA_ENERGIA_DOCUMENTO DOC
INNER JOIN [10.10.1.4].[CORPORE].[dbo].[GFILIAL] FILIAL
  ON FILIAL.ATIVO = 1
 AND REPLACE(REPLACE(REPLACE(FILIAL.CGC, '.', ''), '/', ''), '-', '')
    COLLATE SQL_Latin1_General_CP1_CI_AI = DOC.CUSTOMER_IDENTIFICATION_NUMBER
LEFT JOIN [10.10.1.4].[CORPORE].[dbo].[TMOVCOMPL] TMOVCOMPL
  ON TMOVCOMPL.IDV360 = DOC.ID_V360_DOCUMENTO
 AND TMOVCOMPL.CODCOLIGADA = FILIAL.CODCOLIGADA
```

## Idempotência

Documento:

- `ID_V360_DOCUMENTO`
- `CHAVE_NEGOCIO`

Consumo:

- `ID_DOC`
- `ORDEM_CONSUMO`
